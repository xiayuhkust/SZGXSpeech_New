# Text Processor 文本处理器说明文档

## 功能概述

Text Processor 是一个专门用于处理中文宗教文本的自动化工具，能够将口语化的文本转换为更加规范和正式的书面语形式。

## 处理流程

1. **文本预处理**
   - 编码检测和转换（支持UTF-8和ANSI编码）
   - 文本分段处理（每段最大1000字符）
   - 空白行和多余空格的清理

2. **智能分块处理**
   - 使用段落自然分隔进行文本分块
   - 保持段落完整性，避免在句子中间截断
   - 控制每个块的大小在模型token限制内

3. **GPT处理阶段**
   - 逐块进行处理
   - 保持原文情感和内容完整性
   - 自动标注圣经引用

4. **后处理阶段**
   - 合并处理后的文本块
   - 格式统一化
   - 输出结果文本

## 使用的模型

- **主要模型**：GPT-4
- **用途**：文本优化和规范化
- **特点**：
  - 强大的中文处理能力
  - 对宗教术语的准确理解
  - 保持文本语气和情感的稳定性

## 文本处理规则

### 1. 口语化清理规则
- **强制删除词汇**：
  - 冗余词：这个、那个、然后、就是、呃、啊、对吧
  - 重复强调：非常非常、真的真的、实在是
- **词语合并规则**：
  - "像那个…一样" → "如…一般"
  - "就是说要…" → "要…"
- **保留特例**：
  - 情感强化词：何等、惟独、唯有
  - 呼召用语：你们要听、阿们

### 2. 文本规范化规则
- 修正语法错误
- 保持原文完整性
- 合理分段（每段3-5行）
- 标注圣经引用（和合本标准）

## 性能参数

- 单次处理文本长度：最大20000字
- 处理时间：约3分钟（20000字）
- 支持的文件编码：UTF-8、ANSI
